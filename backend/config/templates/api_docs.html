Getting Started

Core conventions used by every endpoint

Authentication

Send Authorization: Bearer <access_token> for all protected routes. Access tokens expire after 15 minutes.

Pagination

List endpoints return paginated results when supported by the backend. Use ?page and ?page_size when available.

Errors

Errors follow {"error": "message"}. Always check HTTP status codes to handle failures gracefully.

Authentication

/auth/
Method Endpoint Auth Description
POST /auth/register/ Public Create a new customer account.
POST /auth/login/ Public Login and receive access + refresh tokens.
POST /auth/logout/ Required Invalidate the current session tokens.
POST /auth/refresh/ Public Refresh an expired access token.
GET PATCH /auth/profile/ Required Fetch or update customer profile details.
POST /auth/change-password/ Required Update the current password.
GET /auth/check-username/ Public Check if a username is available. Query param: ?username=
GET /auth/check-email/ Public Check if an email is available. Query param: ?email=
GET /auth/check-phone/ Public Check if a phone number is available. Query param: ?phone=

Login example

{ "phone": "9876543210", "password": "your_password" }

Validation response example

{ "available": true, "message": "Phone number is available" }

Products

/products/
Method Endpoint Auth Description
GET /products/ Public List products with filters (category, search, min_price, max_price, in_stock, ordering).
GET /products/{id}/ Public Fetch full details for a single product.
GET /products/featured/ Public Return featured items (rating ≥ 4.5).
GET /products/{id}/related/ Public Get related products from the same category.

Popular filters

  • • ?category=Fruits
  • • ?search=organic
  • • ?min_price=10&max_price=120&ordering=-price

Categories

/products/categories/
Method Endpoint Auth Description
GET /products/categories/ Public List every available category.
GET /products/categories/{id}/ Public Return a single category with metadata.

Shopping Cart

/cart/ Auth required
Method Endpoint Auth Description
GET /cart/ Required Get cart items, subtotal, tax, delivery charge, and total.
POST /cart/add/ Required Add an item (product_id, quantity).
POST /cart/update_item/ Required Update quantity for a cart item (item_id, quantity).
POST /cart/remove_item/ Required Remove a single cart item (item_id).
POST /cart/clear/ Required Remove all items from the cart.

Orders

/orders/ Auth required
Method Endpoint Auth Description
GET /orders/ Required List customer orders (supports status and delivery_type filters).
GET /orders/{id}/ Required Retrieve a single order with items and totals.
POST /orders/ Required Create a new order with delivery details.

Create order payload

{ "items": [ { "product_id": 1, "quantity": 2, "price": 99.99 } ], "delivery_name": "John Doe", "delivery_phone": "9876543210", "delivery_address": "123 Main St", "delivery_type": "standard", "subtotal": 199.98, "tax": 15.99, "delivery_charge": 50, "discount": 0, "total": 265.97, "coupon_code": "WELCOME10" }

Coupons

/coupons/
Method Endpoint Auth Description
GET /coupons/ Public List available coupons.
GET /coupons/{id}/ Public Get a coupon by ID.
POST /coupons/validate/ Public Validate coupon code against current cart.
POST /coupons/apply/ Required Apply a coupon to the authenticated cart.
POST /coupons/remove/ Required Remove the currently applied coupon.
POST /coupons/suggested/ Public Return suggested coupons based on cart totals.

Validate coupon example

{ "code": "WELCOME10", "cart_total": 500, "cart_items": [] }

Wishlist

/wishlist/ Auth required
Method Endpoint Auth Description
GET /wishlist/ Required Return all wishlist items.
POST /wishlist/add/ Required Add product to wishlist (product_id).
POST /wishlist/remove/ Required Remove product from wishlist (product_id).
GET /wishlist/{product_id}/check/ Required Check if a product is saved.
POST /wishlist/clear/ Required Clear the entire wishlist.
